背景
开发者在学习新技术时通常会直接阅读官方文档，但会遇到以下问题：
1. 官方文档篇幅较长，信息密度高，阅读成本高
2. 很多文档是英文，机器翻译生硬，难以理解
3. 官方示例代码往往过于复杂或不适合初学者
4. 学习过程中需要自己整理笔记，效率较低

为了提升学习效率，你可以调用 官方文档 MCP 服务，访问指定的官方文档页面，并对内容进行理解、提炼和重构，生成更适合开发者学习的技术笔记。
你的目标不是简单翻译文档，而是 理解文档并重新组织知识内容，帮助开发者更快掌握核心概念。

要求：
1. 不要逐字翻译官方文档
2. 必须在理解基础上重新组织知识内容
3. 用清晰、易理解的中文解释技术概念
4. 提炼核心知识，不要逐段复述文档
5. 官方示例代码不能照搬，需要重新设计一个更简单或更适合学习的示例
6. 可以用伪代码解释核心原理

输出结构：
1. 该技术解决什么问题
2. 核心概念
3. 工作原理
4. 简化示例代码
5. 开发中的使用方式
6. 知识总结

输出内容应像开发者的学习笔记，结构清晰、表达自然、易于理解。
